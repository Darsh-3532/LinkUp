================================================================================
                           BACKEND API ENDPOINTS
================================================================================

1. GET /api/health
   - Health check endpoint
   - Returns: { success: true, message: "Backend API is running" }

2. GET /api/network
   - Fetch complete network data
   - Returns: { nodes: [...], edges: [...] }

3. GET /api/network/stats
   - Get network statistics
   - Returns: { total_nodes, total_edges, companies, investors, funding_types, date_range }

4. GET /api/filters
   - Get available filter options
   - Returns: { funding_types: [...], date_range, node_types }

5. POST /api/network/filter
   - Filter network by funding type
   - Request: { funding_type: "angel" | "series-a" | ... }
   - Returns: Filtered { nodes: [...], edges: [...], stats }

6. GET /api/search?q=<query>
   - Search nodes by ID or label
   - Returns: Array of matching nodes

7. GET /api/node/:id
   - Get specific node details and connections
   - Returns: { node, connected_nodes, connected_edges }

================================================================================
                            HOW IT WORKS
================================================================================

1. FRONTEND (network_explorer.html)
   - When page loads, fetches /api/network from backend
   - Displays 740 nodes and 400 edges
   - User can select funding type from dropdown
   - Clicking "Apply Filters" calls /api/network/filter endpoint
   - Backend filters edges and returns only connected nodes
   - Frontend re-renders network with filtered data

2. BACKEND (server.js)
   - Loads network data from JSON files
   - Serves static files (HTML, CSS, JS)
   - Provides API endpoints for data queries
   - Performs filtering on the fly
   - Returns filtered network data

3. NETWORK VISUALIZATION
   - Force-directed layout algorithm
   - Blue nodes = Companies
   - Orange nodes = Investors
   - Lines = Investment relationships
   - Node size indicates centrality

================================================================================
                              TROUBLESHOOTING
================================================================================

If Network Explorer shows no nodes:
  1. Verify server is running: npm run server
  2. Check browser console for errors
  3. Confirm http://localhost:5000 is accessible
  4. Verify network_data.json exists in project root
  5. Check that API endpoints respond: curl http://localhost:5000/api/health

If filters don't work:
  1. Check browser console for network errors
  2. Verify POST request to /api/network/filter is successful
  3. Check that fundingTypeFilter element exists in HTML

If server won't start:
  1. Ensure port 5000 is not in use
  2. Run: npm install (to install all dependencies)
  3. Check Node.js version: node --version
  4. Verify network_data.json exists
